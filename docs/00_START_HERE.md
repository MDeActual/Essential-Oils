## Package Purpose

This package defines the architecture, governance, data model, protocol systems, growth systems, trust systems, and business systems for Phyto.ai.

Phyto.ai must be built as:
- a community-driven AI-native natural health discovery platform
- not a generic wellness tracker
- not a supplement-first store
- not an open unmoderated forum
- not a thin AI wrapper

---

## Core Product Priorities

Always prioritize in this order:
1. trust and privacy
2. data integrity
3. protocol engine
4. challenge engine
5. contributor workflows
6. discovery outputs
7. reputation and quality systems
8. growth systems
9. monetization layers

## Latest-Version Rule

Always use the highest version number of a document family.
Do not mix multiple active versions of the same file family.

Example:
- use `phyto_ai_full_blueprint_v7_complete.md`
- ignore older blueprint versions

## Conflict Rule

If files conflict:
- schema wins for data structures
- API wins for service contracts
- UI map wins for screens
- protocol library wins for protocol definitions
- ontology wins for remedy semantics
- blueprint wins for overall architecture intent

## Build Rule

- confirm the authoritative doc exists
