# Cloud Matrix DevOS Foundation

This folder contains the DevOS Foundation v0.1 artifacts used to pilot the Cloud Matrix operating model inside the Essential-Oils repository.

Essential-Oils / Phyto.ai is the first live product used to prove DevOS through real product delivery.

## Foundation Artifacts

| File | Purpose |
|---|---|
| `CLOUD_MATRIX_REFERENCE_ARCHITECTURE_v0.1.md` | Defines the Cloud Matrix / DevOS operating model |
| `CLOUD_MATRIX_CONSTITUTION.md` | Defines durable Cloud Matrix operating principles |
| `DEVOS_COMMAND_CENTER.md` | Daily front door for priorities, decisions, blockers, and next actions |
| `AGENT_CHARTER_TEMPLATE.md` | Standard template for creating specialized Cloud Matrix agents |
| `DEVOS_ALIGNMENT_PR_CHECKLIST.md` | Pull request alignment standard for DevOS-governed repositories |
| `DECISION_ENGINE_CONTRACT.md` | Defines how DevOS turns context into decisions |
| `METHODOLOGY_CHANGELOG.md` | Tracks changes to how Cloud Matrix builds software |

## Operating Intent

DevOS should reduce cognitive load and improve decision quality.

It should not create documentation bloat.

Each artifact should either:

1. define a principle,
2. define a process, or
3. record a decision.

## Pilot Rule

Essential-Oils is the pilot.

Patterns that work here can later be promoted into reusable Cloud Matrix / DevOS standards for SecurePulse and future products.
